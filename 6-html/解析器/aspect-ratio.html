在 CSS 中，有一个新的属性叫做 aspect-ratio（长宽比）属性，它可以用来设置元素的长宽比。这个属性在 CSS Contain 布局中非常有用，它允许你在不同的布局环境下保持元素的固定比例。
在 CSS 中，有许多不同的单位可以用来表示长度、宽度、高度、字体大小等。这些单位可以帮助你在网页设计和布局中精确控制元素的尺寸和样式。以下是一些常见的 CSS 单位： 像素（px）：像素是最常见的单位，表示屏幕上的一个物理像素。它是固定的，不会随视窗或屏幕大小变化而变化。 百分比（%）：百分比单位是相对于父元素的尺寸来计算的。例如，width: 50% 表示元素宽度为父元素宽度的 50%。 视窗单位（vw、vh、vmin、vmax）：视窗单位是相对于视窗尺寸的单位。vw 表示视窗宽度的百分比，vh 表示视窗高度的百分比，vmin 和 vmax 分别表示视窗宽度和高度的较小或较大值。 相对单位（em、rem）：em 是相对于元素的字体大小计算的单位，rem 是相对于根元素（）的字体大小计算的单位。它们可以用于字体大小、内外边距等。 毫米（mm） 和 厘米（cm）：用于打印和其他物理介质的单位，基于物理尺寸。 英寸（in）：也用于打印和物理介质，基于英寸尺寸。 点（pt）：通常用于印刷，1 英寸约为 72 点。 pica（pc）：通常用于印刷，1 英寸约为 6 pc。 这些是一些常见的 CSS 单位，它们在不同的情况下可以灵活地用于调整元素的样式和布局。在选择单位时，需要根据具体的设计需求和布局情况来选择合适的单位。
使用 rem 单位进行移动端适配具有一些优点和缺点，让我们来看看： 优点： 相对于根元素大小：rem 单位是相对于根元素（）的字体大小计算的，因此可以在不同设备和分辨率下保持一致的相对比例，从而实现良好的适配效果。 一致的尺寸调整：通过调整根元素的字体大小，可以一次性地调整页面中所有元素的尺寸，而不需要逐个调整。这对于响应式设计和布局非常有用。 适合弹性布局：rem 单位适用于弹性布局，当页面需要在不同屏幕尺寸和设备上进行自适应时，它可以帮助你实现弹性和流式的布局。 支持缩放：由于 rem 单位会随着根元素字体大小的调整而改变，页面具有一定程度的缩放和放大能力。 缺点： 不稳定的基准字体大小：在一些情况下，可能会遇到根元素字体大小不稳定的问题，特别是在某些浏览器或系统设置中用户可能修改了默认字体大小。 不支持老旧浏览器：一些较老的浏览器可能不完全支持 rem 单位，可能需要使用 px 或其他单位进行降级处理。 复杂的计算：在某些情况下，使用 rem 单位进行计算可能会稍显复杂，特别是当需要处理复杂的嵌套关系和动态布局时。 适配上限：在某些情况下，使用 rem 单位可能无法实现某些特定的适配需求，比如需要更精细的控制或与其他适配方案结合使用。 总体来说，rem 单位是一种强大的工具，适用于大多数移动端适配场景。然而，要充分理解其优点和缺点，并在项目中根据实际需求进行合理选择和使用。
